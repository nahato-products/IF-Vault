# Qiita記事執筆システム

Obsidian + Claude Codeで、**唯一無二かつAIっぽくない**Qiita記事を執筆するための統合システムです。

## 🎯 このシステムの特徴

- **あなたの生の声を引き出す** - インタビュー形式で体験・感情を記事化
- **AIっぽい文章を排除** - 堅苦しい表現を検出し、カジュアルな文体に修正
- **記事候補を自動発見** - Daily NotesやArchivesから記事ネタを探索
- **完全統合ワークフロー** - 下書き→推敲→投稿まで一貫してサポート

## 📁 ディレクトリ構造

```
11_Qiita/
├── drafts/           # 執筆中の記事
├── published/        # 公開済みの記事
├── templates/        # 記事テンプレート
├── .qiita-config.yaml # 文体チェック設定
└── README.md         # このファイル
```

## 🚀 使い方

### 🎯 推奨: ワークフロー管理を使う（NEW！）

**たった1つのコマンドで、すべてを自動案内:**

```bash
/qiita-workflow start
```

あとは提案に「はい」と答えるだけ。次に何をすべきか自動的に案内されます！

**中断した記事を再開:**
```bash
/qiita-workflow continue
```

**進捗状況を確認:**
```bash
/qiita-workflow
```

詳しくは **`WORKFLOW_GUIDE.md`** をご覧ください。

---

### または、手動でステップを実行:

### ステップ0: 記事ネタを探す📚

```bash
/qiita-topics-from-slack
```

**何が起きるか:**
1. Slackの特定チャンネル（dev、randomなど）から記事ネタを探索
2. 技術的な会話、トラブル解決、勉強会参加などを自動抽出
3. 記事候補を3-5個提示
4. 気になるトピックで記事執筆を開始

### ステップ1: 記事の下書きを作成

```bash
/qiita-draft
```

または、トピックを指定:

```bash
/qiita-draft tmuxの使い方
```

**何が起きるか:**
1. Daily NotesやArchivesから記事候補を探索（トピック未指定時）
2. インタビュー形式であなたの体験・感情を引き出す
3. カジュアルで人間味のある文章で下書きを作成
4. `11_Qiita/drafts/` に保存

### ステップ2: 文体チェック・推敲

```bash
/qiita-review
```

または、特定の記事を指定:

```bash
/qiita-review 11_Qiita/drafts/2026-02-17_tmuxの使い方.md
```

**何が起きるか:**
1. AIっぽい表現（「〜について説明します」など）を検出
2. カジュアルな表現への修正案を提示
3. 体験・感情の記述があるかチェック
4. 自動修正または手動修正を選択可能

### ステップ3: Qiitaに投稿

```bash
/qiita-publish
```

**何が起きるか:**
1. フロントマターを削除し、Qiita投稿用フォーマットに変換
2. Obsidianリンクを通常のテキストに変換
3. **【NEW！】Claude Codeから直接投稿**、またはコピペ用の本文を表示 🚀
4. 投稿後、`published/` に自動移動、URL自動記録

## ⚙️ 設定ファイル

### `.qiita-config.yaml`

AIっぽい表現のパターンや、推奨する文体を定義しています。

**カスタマイズ可能な項目:**
- `ai_like_patterns` - 検出するNGパターン
- `recommended_expressions` - 推奨する表現
- `source_directories` - 記事ネタを探すディレクトリ
- `interview_questions` - インタビューの質問テンプレート

## 📝 記事テンプレート

`templates/article-template.md` に記事の基本構造が定義されています。

**フロントマター:**
```yaml
---
title: "記事タイトル"
tags: [タグ1, タグ2, タグ3]
status: draft  # draft または published
created: 2026-02-17
qiita_url: ""
---
```

## 🎨 文体のルール

### ❌ 避けるべき表現（AIっぽい）

- 「〜について説明します」→ 「〜を見ていきます」
- 「〜することができます」→ 「〜できます」
- 「本記事では」→ 「今回は」「この記事では」
- 「以下の通りです」→ 「こんな感じです」
- 「ご覧ください」→ 「見てみてください」

### ✅ 推奨する表現（カジュアル・人間味）

- 「〜してみました」
- 「〜だと思います」
- 「〜ですね」
- 「ちょっと〜」「けっこう〜」「実際〜」
- 「個人的には〜」「正直〜」
- 「躓いた」「ハマった」「嬉しかった」

Claude Codeから直接Qiitaに投稿できます。

**セットアップ:**
1. `cd 11_Qiita/qiita-mcp-server && npm install`
2. Qiitaアクセストークンを取得

4. `~/.claude/config.json` にMCPサーバーを登録
5. Claude Code再起動

詳しくは `11_Qiita/qiita-mcp-server/README.md` を参照。

## 📚 記事のネタ源

このシステムは、以下のディレクトリから記事候補を自動で探索します:

- `05_Daily Notes` - 日々の作業ログや学習メモ
- `04_Archives` - イベント参加レポートや技術メモ
- `06_Knowledge` - 学習記録
- `01_Projects` - プロジェクト関連のトラブルシューティング

## 🎯 良い記事を書くコツ

1. **体験を具体的に書く** - 「困った」「嬉しかった」などの感情を含める
2. **失敗談を恐れない** - ハマったポイントは貴重な情報
3. **完璧を目指さない** - ラフで読みやすい文章を心がける
4. **あなたの言葉で書く** - 一般論ではなく、個人の意見・感想を書く
5. **具体例を入れる** - 抽象的な説明だけでなく、実際の例を示す
