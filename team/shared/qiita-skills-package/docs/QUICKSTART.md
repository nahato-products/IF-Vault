## 🚀 最初の記事を書いてみよう

### ステップ1: 記事候補を探す（2分）

Claude Codeで以下のコマンドを実行:

```
/qiita-draft
```

**何が起こるか:**
1. Daily NotesやArchivesから記事候補を探します
2. 3-5個のトピック候補が提示されます
3. 気になるトピックを選択します

**例:**
```
以下の記事候補が見つかりました:

1. tmuxの使い方とセットアップ (05_Daily Notes/2026-02-10.md)
2. OceanBase Meetup参加レポート (04_Archives/...)
3. 統計学イベントの学び (04_Archives/...)

どの記事を書きますか？
```

### ステップ2: インタビューに答える（3分）

トピックを選ぶと、Claude Codeが質問してきます。

**質問例:**
- この技術と最初に出会ったきっかけは？
- 実際に試してみてどう感じましたか？
- 躓いたポイントはありましたか？
- どうやって解決しましたか？

**コツ:**
- **正直に答える** - 失敗談も大歓迎
- **感情を入れる** - 「困った」「嬉しかった」「意外だった」など
- **具体的に話す** - 「けっこう大変だった」より「3時間ハマった」

### ステップ3: 下書きを確認（1分）

Claude Codeが下書きを作成してくれます。

**保存場所:**
```
11_Qiita/drafts/2026-02-17_tmuxの使い方.md
```

Obsidianで開いて内容を確認しましょう。

### ステップ4: 文体チェック（2分）

```
/qiita-review
```

**何が起こるか:**
1. AIっぽい表現（「〜について説明します」など）を検出
2. カジュアルな表現への修正案を提示
3. 自動修正するか選択

**例:**
```
🔴 修正必須: 3箇所

1. [12行目] "tmuxについて説明します"
   → "tmuxを見ていきます" に修正を推奨

修正を実施しますか？
1. 自動で修正する
2. 手動で修正する
```

### ステップ5: Qiitaに投稿（3分）

```
/qiita-publish
```

**何が起こるか:**
1. Qiita投稿用フォーマットに変換
2. コピペ用の本文を表示

**投稿方法:**
1. [Qiita新規投稿ページ](https://qiita.com/drafts/new)を開く
2. 生成された本文をコピペ
3. タグを設定
4. 「限定共有で投稿」または「公開」をクリック

## 💡 次のステップ

### より良い記事を書くために

1. **Daily Notesを習慣化**
   - 毎日の学習や作業ログを書く
   - 躓いたポイントを記録する
   - イベント参加したら感想を書く

2. **定期的にレビュー**
   - 週に1回、Daily Notesを見返す
   - 記事にできそうなネタをピックアップ
   - `/qiita-draft` で下書き作成

3. **文体を磨く**
   - `.qiita-config.yaml` をカスタマイズ
   - 自分らしい表現を「推奨表現」に追加
   - NGパターンを追加

### カスタマイズ例

#### 独自のNGパターンを追加

`11_Qiita/.qiita-config.yaml` を編集:

```yaml
ai_like_patterns:
  # デフォルトのパターン...

  # 自分専用のNGパターンを追加
  - pattern: "〜と考えられます"
    suggestion: "〜だと思います"
  - pattern: "〜ということです"
    suggestion: "〜ってことです"
```

#### インタビュー質問をカスタマイズ

```yaml
interview_questions:
  # デフォルトの質問...

  # 自分専用の質問を追加
  - "この技術を誰かに勧めたい？勧めたくない？"
  - "もし最初からやり直すなら、何を変える？"
```
