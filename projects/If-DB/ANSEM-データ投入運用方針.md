# ANSEM IF-DB データ投入運用方針

## 背景

既存のインフルエンサーマスタは **データが汚すぎて整合性が取れない** 状態。クレンジング（既存データの修正）では対応しきれないため、既存データの流用は断念し、**新DBにゼロから入れ直す** 方針に決定。これは「移行」ではなく「再登録」。

## 基本方針

各エージェント担当者が、自分の担当インフルエンサーの情報を投入する

- 案件発生時/契約時に必要な情報を登録
- 担当者が自らデータ品質に責任を持つ
- 正確なデータのみをDBに蓄積

## データ投入方法

運用の柔軟性を確保するため、2つの投入方法を用意する（詳細は [[ANSEM-一括登録機能メモ]] 参照）

### 1. 個別登録（通常運用）

- 管理画面のフォームから1件ずつ登録
- 案件ごとにインフルエンサー情報を入力
- リアルタイムでDB反映
- 入力時にバリデーションが効く

> [!TIP] こんな時に使う
> - 新規案件で初めて取引するインフルエンサー
> - 契約時に詳細情報を確実に登録したい
> - 少数のデータを丁寧に入れたい

### 2. 一括登録（効率化）

- Googleスプレッドシート/Excelで準備してアップロード
- 自分のペースで手元のスプシで入力・整理できる
- まとめて投入して効率化

> [!TIP] こんな時に使う
> - 既に管理している複数のインフルエンサーをまとめて投入
> - 過去案件のデータを整理して登録
> - 大量のデータを一気に入れたい

## データ投入のタイミング

| タイミング    | 投入データ         | 方法   |
| -------- | ------------- | ---- |
| 案件打診時    | 基本情報、SNSアカウント | 個別登録 |
| 契約締結時    | 報酬条件、銀行口座、住所  | 個別登録 |
| 過去データ整理時 | 担当インフルエンサー情報  | 一括登録 |
| 情報更新時    | 変更された項目のみ     | 個別登録 |

## データ品質の保証

### 投入前チェック

- 重複登録の防止（システムで自動チェック）
- 必須項目の入力確認
- メールアドレス、電話番号等のフォーマットチェック

### 投入後の記録

- 誰が・いつ登録したか自動記録（`created_by` / `updated_by`）
- データ変更履歴の保持
- 担当エージェントの明確化

## 運用ルール

### データ所有

- 登録したエージェントがそのデータの責任者
- 他エージェントのデータは原則編集不可
- 担当変更時は管理者が移管処理

### データ更新

- インフルエンサー情報変更時は即座に更新
- 古い情報のまま放置しない
- 報酬条件等の変更も都度記録
