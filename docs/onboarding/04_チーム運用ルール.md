# チーム運用ルール

Vault をチームで使う上でのルール、よくある質問、トラブルシューティングをまとめた。

---

## やること

### 1. 自分のフォルダで作業する

`team/あなたの名前/` 配下で自由に作業する。他のメンバーのフォルダは見るだけ。

### 2. 毎日デイリーノートを書く

- 作業記録を残す
- 学んだことを記録する
- 明日の予定を書く

### 3. わかりやすいファイル名を付ける

「無題のファイル.md」はダメ。「プロジェクトX進捗報告.md」のように、開かなくても内容がわかる名前にする。

### 4. タグを活用する

- `#自分の名前` - 自分のノート
- `#project` - プロジェクト関連
- `#meeting` - ミーティング記録

### 5. 内部リンクでノートをつなぐ

`[[関連ノート]]` で情報を有機的に整理する。孤立したノートはもったいない。

### 6. 共有ファイルの編集前に Pull する

`Cmd + P` → "Git: Pull" で最新を取得してから編集する。競合防止のため。

---

## やらないこと

### 1. 他メンバーのフォルダを編集しない

参照・リンクは OK。中身を変えたいときは本人に相談する。

### 2. 機密情報を書かない

パスワード、API キー、個人情報は Vault に入れない。`.gitignore` で除外設定はしてあるが、手動で書かないこと。

### 3. 大きなファイルをコミットしない

画像は 10MB 以下に抑える。動画は Vault に入れない。大きなファイルは Google Drive など別の場所で管理する。

### 4. Git を無効化しない

Obsidian Git プラグインは常に有効にしておく。自動バックアップとチーム同期の生命線。

---

## タスク管理

タスクは基本的に Notion で管理する。Vault のデイリーノートには作業記録を書くが、タスクの発行・管理は Notion 側で行う。

- プロジェクトのタスク（WBS）→ Notion のプロジェクトページ
- 個人 TODO → Notion または Obsidian の Kanban
- 全社・チーム共通タスク（マスト） → Notion のチームページ

タスクを誰かに依頼するときは、Notion に記載して本人に伝える。

## よくある質問

### Obsidian が起動しない

アプリケーションフォルダに入っているか確認する。入っていれば再インストールを試す。

### Git が動かない

`Cmd + P` → "Git: Initialize a new repo" を試す。それでもダメなら SSH 鍵が正しく設定されているか確認。

```bash
ssh -T git@github.com
# "Hi username!" と表示されれば OK
```

### プラグインが表示されない

1. Obsidian を再起動
2. 設定 → コミュニティプラグイン → 制限モードがオフになっているか確認

### Git の競合が発生した

競合ファイルを開くと `<<<<<<<` と `>>>>>>>` で囲まれた箇所がある。自分の変更と相手の変更を見比べて、正しい方を残す。

```bash
cd ~/Documents/Obsidian\ Vault
git status                            # 競合ファイルを確認
# ファイルを手動で編集して解決
git add .
git commit -m "Merge conflict resolved"
git push
```

自力で解決できなければ先輩に相談する。

### 自動同期が動作しない

設定 → コミュニティプラグイン → Obsidian Git を開いて確認:

- 「Auto backup after file change」が有効か
- 「Auto save interval」が 10 に設定されているか

### 誤ってファイルを削除した

Git 履歴から復元できる。先輩に相談すれば戻してもらえる。

### 他メンバーのノートを見たい

`team/メンバー名/` を開いて参照する。見るのは自由、編集は本人だけ。

### スマホで見たい

GitHub の Web ページから閲覧可能。
https://github.com/nahato-Inc/IF-Vault

### 会社と自宅の PC で同期したい

両方の PC でリポジトリをクローンする。作業前に `Cmd + P` → "Git: Pull" で最新を取得してから作業する。

### Claude Code のエージェントが動作しない

セットアップスクリプトを再実行する。

```bash
./claude-agents-setup.sh
```

解決しなければ guchi に相談。
