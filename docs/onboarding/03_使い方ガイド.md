# 使い方ガイド

Obsidian の日常的な操作方法をまとめたガイド。セットアップが終わった後に読む。

---

## 毎日の作業フロー

### 朝（業務開始時）

1. Obsidian を起動
2. `Cmd + P` → "Open today's daily note" でデイリーノートを開く
3. 今日のタスクと目標を記入

### 作業中

- 会議 → `meeting-note-template` を使って記録
- コード → `code-snippets/` に保存
- 学んだこと → `notes/` に記録
- プロジェクト → `projects/` で管理

メモは随時書く。10分ごとに自動で GitHub にバックアップされるので、保存を意識する必要はない。

### 終業前

1. デイリーノートに作業記録を追記
2. 明日の予定を書く
3. 手動で即座に保存したい場合は `Cmd + P` → "Git: Commit all changes"

## テンプレートの使い方

4種類のテンプレートが用意されている。

### デイリーノート

`Cmd + P` → "Open today's daily note" で自動適用される。手動で挿入する場合は `Cmd + P` → "Templater: Insert template" → `daily-note-template` を選択。

テンプレートの中身:

```
# YYYY-MM-DD (曜日)

## 今日のタスク
- [ ]

## メモ
## 学んだこと
## 作業記録
### 午前
### 午後
## 明日の予定
```

### 会議メモ

`Cmd + P` → "Templater: Insert template" → `meeting-note-template`

基本情報（日時・参加者・場所）、アジェンダ、議事内容、決定事項・アクションアイテムの構成。

### プロジェクト

`Cmd + P` → "Templater: Insert template" → `project-template`

目標、スケジュール、タスクリスト、進捗の構成。

### コードスニペット

`Cmd + P` → "Templater: Insert template" → `code-snippet-template`

概要、コード、説明、使い方、参考リンクの構成。

## Git 操作

### 自動バックアップ（設定済み）

10分ごとに自動で GitHub に保存される。何もしなくてOK。

### 手動保存

すぐに保存したいとき:

- `Cmd + P` → "Git: Commit all changes" → コミット
- `Cmd + P` → "Git: Push" → GitHub に送信

### 同期（他の PC の変更を取得）

- `Cmd + P` → "Git: Pull" → 最新の状態に更新

作業前に Pull しておくと競合を防げる。

## コード実行

Execute Code プラグインで、ノート内にコードを書いて実行できる。

### Python

````markdown
```python
print("Hello, Obsidian!")
```
````

### JavaScript

````markdown
```javascript
console.log("Hello, World!");
```
````

コードブロックの右上に表示される「▶️」ボタンで実行。

## GitHub Alert 記法

```markdown

> 青い枠の注記

> [!TIP]
> 緑の枠のヒント

> 黄色い枠の警告

> [!IMPORTANT]
> 紫の枠の重要事項

> [!CAUTION]
> 赤い枠の注意
```

## Dataview でノート検索

Dataview プラグインを使うと、SQL ライクなクエリでノートを検索・集計できる。

### 自分の最近のノート10件

```dataview
table file.mtime as 更新日時
from "team/あなたの名前"
sort file.mtime desc
limit 10
```

### 今週作成したノート

```dataview
table file.cday as 作成日
from "team/あなたの名前"
where file.cday >= date(today) - dur(7 days)
```

### タグで検索

```dataview
list
from #あなたの名前
```

`"team/あなたの名前"` の部分を自分のフォルダ名に置き換えて使う。

## リンクの使い方

### 内部リンク

ノート同士をリンクでつなぐと、関連する情報に素早くアクセスできる。グラフビューで全体像も見える。

```markdown
[[ノート名]]
[[ノート名|表示テキスト]]
[[ノート名#見出し名]]
```

### グラフビュー

`Cmd + P` → "Open graph view" でノート間のつながりを可視化できる。

### バックリンク

右サイドバーに、今開いているノートにリンクしている他のノートが表示される。「このノートはどこから参照されているか」を確認するのに使う。

## Kanban ボードでタスク管理

1. 新規ノートを作成
2. `Cmd + P` → "Kanban: Create new board"
3. カラムを作成（例: Not Started / In Progress / Done）
4. カードをドラッグ&ドロップで移動
